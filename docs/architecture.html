Request Data Flow

User changes a filter

FilterBar.vue updates the useFilters singleton ref (e.g., selectedStatus = "Delivered"). All views share this state via module-level refs.

View detects change and calls API

A watch on the filter ref triggers the view to call api.getOrders(getCurrentFilters()). The composable builds { warehouse, category, status, month } from current state.

api.js sends HTTP request

Axios serializes filters into query params and sends GET /api/orders?status=Delivered to the FastAPI backend at localhost:8001.

FastAPI handler processes request

The route handler extracts query params, calls apply_filters(orders, ...) and optionally filter_by_month(). Pydantic validates each result against the Order model.

In-memory filtering

Python list comprehensions scan the in-memory data loaded from JSON. Filters are case-insensitive; values of "all" skip that filter. Quarter strings like Q1-2025 expand to their three months.

JSON response returned

FastAPI serialises the validated list to JSON and returns it with a 200 OK. Pydantic enforces the response model shape before sending.

Vue updates reactive state and re-renders

The view stores the result in its local ref(). Vue's reactivity system triggers a re-render; computed properties (e.g., ordersByStatus) recalculate automatically.

Filter System

  • param Time Period — month (2025-01) or quarter (Q1-2025)
  • param Warehouse — San Francisco, London, Tokyo
  • param Category — Circuit Boards, Sensors, Actuators ...
  • param Order Status — Delivered, Shipped, Processing, Backordered

All four filters are combined as query params on every API call. Inventory ignores the month param (no time dimension).

Core Data Models

InventoryItem

  • id, sku, name
  • category — Circuit Boards | Sensors | Actuators | Controllers | Power Supplies
  • warehouse — San Francisco | London | Tokyo
  • quantity_on_hand, reorder_point
  • unit_cost, location, last_updated

Order

  • id, order_number, customer
  • items[] — sku, name, quantity, unit_price
  • status — Delivered | Shipped | Processing | Backordered
  • warehouse, category
  • order_date, expected_delivery, actual_delivery
  • total_value

DemandForecast

  • item_sku, item_name
  • current_demand, forecasted_demand
  • trend — increasing | stable | decreasing
  • period — e.g., "Next 30 days"

BacklogItem & PurchaseOrder

  • order_id, item_sku, item_name
  • quantity_needed, quantity_available
  • days_delayed, priority — high | medium | low
  • has_purchase_order — computed flag
  • PO: supplier_name, quantity, unit_cost, expected_delivery_date, status
