System Architecture Overview

Tech Stack

Frontend
Vue 3 + Vite
Port 3000
  • Composition API (setup pattern)
  • Vue Router 4 (history mode)
  • Axios for HTTP requests
  • Custom SVG charts (no library)
  • Scoped CSS / CSS Grid layouts
Backend
Python FastAPI
Port 8001
  • Uvicorn ASGI server
  • Pydantic v2 data validation
  • CORS middleware (open in dev)
  • Auto-generated OpenAPI docs
  • uv package manager
Data Layer
JSON + In-Memory
No database
  • 7 JSON files in server/data/
  • Loaded once at server startup
  • In-memory filtering (Python lists)
  • Changes don't persist across restarts
  • Pydantic models enforce schema

Views & API Endpoints

Frontend Routes (Vue Router)

  • /Dashboard - KPIs, charts, summary
  • /inventoryStock levels by warehouse
  • /ordersOrder tracking + status
  • /demandForecast trends
  • /spendingBudget analytics
  • /reportsQuarterly + monthly trends

Backend API Endpoints

  • GET /api/inventorywarehouse, category
  • GET /api/orderswarehouse, category, status, month
  • GET /api/dashboard/summaryAll filters
  • GET /api/demandNo filters
  • GET /api/backlogNo filters
  • GET /api/spending/*summary, monthly, categories, txns
  • GET /api/reports/*quarterly, monthly-trends

Shared State (Composables)

Vue composables use the singleton pattern - refs are declared outside the function so all components share the same reactive state.

useFilters()
4 filter refs (period, location, category, status) + getCurrentFilters() for API calls
useAuth()
User authentication state shared across ProfileMenu and other components
useI18n()
Internationalization with English and Japanese locale files

Backend Data Models (Pydantic)

Core Models

  • InventoryItemid, sku, name, category, warehouse, quantity, reorder_point, unit_cost
  • Orderid, order_number, customer, items[], status, order_date, total_value
  • DemandForecastid, item_sku, current_demand, forecasted_demand, trend
  • BacklogItemid, order_id, item_sku, quantity_needed, days_delayed, priority

Supporting Models

  • PurchaseOrderid, backlog_item_id, supplier, quantity, unit_cost, status
  • CreatePurchaseOrderRequestRequest body for POST /api/purchase-orders

Filter Functions

  • apply_filters()Generic warehouse/category/status filter
  • filter_by_month()Month + quarter (Q1-2025) date filter
