Global Filter System

useFilters Composable

Singleton module — all views share the same reactive state.

  • selectedPeriod — month (2025-01) or quarter (Q1-2025) or "all"
  • selectedLocation — San Francisco / London / Tokyo / "all"
  • selectedCategory — Circuit Boards / Sensors / etc. / "all"
  • selectedStatus — Delivered / Shipped / Processing / Backordered / "all"

Backend Filter Logic

Filters applied sequentially on in-memory copies (non-mutating).

  • Warehouse: exact match
  • Category: case-insensitive match
  • Status: exact match
  • Month: direct (2025-01) or quarter expansion (Q1 = 01, 02, 03)
  • Inventory endpoints do not support month filter

Key Architectural Patterns

Singleton Filter State

The useFilters composable is a module-level singleton. All views import and watch the same reactive refs — no prop drilling or Vuex needed.

Raw refs + Computed

Raw API data lives in ref(). All derived values (KPIs, charts, aggregates) are computed() properties that update reactively when raw data changes.

Backend Filtering

All filtering happens server-side. Frontend passes query params; backend filters in-memory arrays non-mutably and returns the result.

Pydantic Validation

Every API response is validated through a Pydantic model, ensuring type safety and consistent JSON structure across all endpoints.

i18n + Currency

Language toggle switches EN/JA text and USD/JPY currency (1:150 rate). Field values like names and warehouses also have translated variants.

Modal Detail Views

Clicking any table row opens a detail modal. Modals receive item data via props and emit close events — no routing required.
