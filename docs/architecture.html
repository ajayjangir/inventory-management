System Architecture

Data Flow — Filter Propagation

1

FilterBar.vue

User selects period, location, category, or status from dropdowns

→
2

useFilters()

Singleton composable updates shared reactive refs (module-level state)

→
3

View watches

Each view watches filter refs and calls getCurrentFilters() on change

→
4

api.js

Builds URLSearchParams, skipping "all" values. Sends GET to backend

→
5

FastAPI

apply_filters() + filter_by_month() filter in-memory data, return JSON

Data Entities (Pydantic Models)

InventoryItem

  • id, sku, name: str
  • category, warehouse, location: str
  • quantity_on_hand, reorder_point: int
  • unit_cost: float
  • last_updated: str

Order

  • id, order_number, customer: str
  • items: List[dict]
  • status, order_date, expected_delivery: str
  • total_value: float
  • warehouse?, category?, actual_delivery?: Optional[str]

DemandForecast

  • id, item_sku, item_name: str
  • current_demand, forecasted_demand: int
  • trend, period: str

BacklogItem

  • id, order_id, item_sku, item_name: str
  • quantity_needed, quantity_available: int
  • days_delayed: int
  • priority: str
  • has_purchase_order?: Optional[bool]

PurchaseOrder

  • id, backlog_item_id, supplier_name: str
  • quantity: int
  • unit_cost: float
  • expected_delivery_date, status: str
  • created_date: str
  • notes?: Optional[str]

Spending (untyped)

  • spending_summary: dict
  • monthly_spending: list
  • category_spending: list
  • recent_transactions: list
