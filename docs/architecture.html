Tech Stack
⚡
Frontend
Vue 3

Composition API, Vite build tooling, Vue Router (hash-free history mode), no Vuex/Pinia — composable singletons for state.

:3000
🐍
Backend
FastAPI

Python 3 async API. Pydantic v2 models for request/response validation. CORS enabled for local cross-origin calls. Managed by uv.

:8001
📦
Data Layer
In-Memory JSON

No database. Seven JSON files in server/data/ are loaded at startup into Python lists. Mutations (POST) are in-memory only and reset on restart.

🔗
HTTP Client
Axios

Centralised API client in client/src/api.js. All 17 exported functions funnel through a single Axios instance pointing to localhost:8001.

🌍
i18n + Currency
Custom Composable

English / Japanese toggle. Locale persisted to localStorage. USD ↔ JPY at 150:1 fixed rate via currency.js.

🧪
Testing
Pytest

Backend tests via FastAPI TestClient in tests/backend/. Covers dashboard, inventory, and miscellaneous endpoints.

Filter Scope — What Gets Filtered

Endpoint warehouse category status month Notes
/api/inventory Yes Yes No No No time dimension on stock
/api/orders Yes Yes Yes Yes Full filter support
/api/dashboard/summary Yes Yes Yes Yes Backlog count always unfiltered
/api/demand No No No No Static forecast data
/api/backlog No No No No Static shortage list
/api/spending/* No No No No Pre-aggregated static data
Data Flow — Example: Dashboard on Filter Change
01
User Interaction

User selects "London" in the Location dropdown in FilterBar.vue

02
Ref Update

The module-scope selectedLocation ref in useFilters.js updates. All watchers notified.

03
Watch Fires

Dashboard.vue watch on filter refs triggers loadData() immediately.

04
Parallel Fetch

Promise.all() fires 4 concurrent requests:
summary, orders, inventory, backlog.

?warehouse=London
05
FastAPI Filters

apply_filters() chains warehouse → category → status. filter_by_month() handles YYYY-MM and Q#-YYYY strings.

06
JSON Response

Pydantic validates and serialises the filtered Python lists to JSON. Axios resolves each Promise.

07
Refs Updated

Raw data refs (allOrders, inventoryItems, etc.) are assigned the new response data.

08
Computed Re-run

All computed properties auto-recalculate. Vue re-renders charts, tables, and KPI cards with London-scoped data.

API Endpoints
Method Path Query Params Description
GET /api/inventory warehousecategory All inventory items, optionally filtered
GET /api/inventory/{id} — Single inventory item by ID
GET /api/orders warehousecategorystatusmonth All orders with full filter support
GET /api/orders/{id} — Single order by ID
GET /api/dashboard/summary warehousecategorystatusmonth Aggregated KPIs: inventory value, low stock, pending orders, backlog count, orders value
GET /api/demand — Demand forecasts (unfiltered)
GET /api/backlog — Backlog shortage items with has_purchase_order flag injected
GET /api/spending/summary — Procurement spending summary totals
GET /api/spending/monthly — Monthly spending breakdown (Jan–Dec)
GET /api/spending/categories — Spending by category
GET /api/spending/transactions — Recent individual transactions
GET /api/reports/quarterly — Aggregated quarterly stats from orders
GET /api/reports/monthly-trends — Month-by-month order count and revenue
GET /api/tasks — Task list
POST /api/tasks — Create a new task
PATCH /api/tasks/{id} — Toggle task completion
DELETE /api/tasks/{id} — Delete a task
POST /api/purchase-orders — Create purchase order for a backlog shortage
GET /api/purchase-orders/{backlog_id} — Retrieve PO by associated backlog item ID
