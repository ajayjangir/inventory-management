System Architecture

Browser — Vue 3 SPA (port 3000)
  • Dashboard.vue KPIs, order health donut chart
  • Inventory.vue Stock table, search, detail modals
  • Orders.vue Order list, status badges, item dropdowns
  • Demand.vue Forecast table, trend indicators
  • Spending.vue Revenue vs costs, monthly charts
  • Reports.vue Quarterly table, monthly trends bar chart
  • Backlog.vue Shortage items, priority levels
↓ HTTP / Axios
FastAPI (port 8001)
  • /api/inventory warehouse · category filters
  • /api/orders warehouse · category · status · month filters
  • /api/dashboard/summary all 4 filters, computed totals
  • /api/demand & /api/backlog no filters
  • /api/spending/* summary · monthly · categories · transactions
  • /api/reports/* quarterly · monthly-trends
  • /api/tasks & /api/purchase-orders CRUD
↓ In-Memory
Data Layer — mock_data.py + server/data/*.json
  • inventory.json ~80 SKUs, 5 categories, 3 warehouses
  • orders.json ~1 300 orders, Jan–Sep 2025
  • demand_forecasts.json 9 items with trend direction
  • backlog_items.json 4 shortage records
  • spending.json 12-month cost breakdown by 4 cost types
  • transactions.json & purchase_orders.json

Data Flow — Filter Request Lifecycle

FilterBar.vue
User selects period / warehouse / category / status
→
useFilters()
Singleton reactive state shared across all views
→
api.js
Builds URLSearchParams, Axios GET request
→
FastAPI route
Receives query params as typed arguments
→
apply_filters()
Immutable list comprehension filtering in Python
→
Pydantic
Validates & serialises response models
→
computed()
Vue derived state applies local search on top

Filter System

Time Period
all · 2025-01 … 2025-09 · Q1-2025 … Q4-2025
Warehouse
all · San Francisco · London · Tokyo
Category
all · Circuit Boards · Sensors · Actuators · Controllers · Power Supplies
Order Status
all · Processing · Shipped · Delivered · Backordered

Inventory endpoint ignores the month filter (no time dimension on stock data). Quarter strings are mapped to individual months on the backend via QUARTER_MAP.

Frontend Composables

useFilters.js
  • Singleton — shared mutable refs across all views
  • selectedPeriod, selectedLocation, selectedCategory, selectedStatus
  • getCurrentFilters() → query param object
  • resetFilters(), hasActiveFilters computed
useI18n.js
  • currentLocale (en / ja), currentCurrency (USD / JPY)
  • t(key, params) — nested lookup with EN fallback
  • translateProductName(), translateWarehouse()
  • Intl API for currency/number formatting
useAuth.js
  • Mock user — name, jobTitle, department, location
  • Tasks array hardcoded per locale
  • Locale-aware field values

API Endpoints

GET/api/inventory
All inventory items
?warehouse & ?category
GET/api/inventory/{id}
Single inventory item by ID
GET/api/orders
All orders
?warehouse & ?category & ?status & ?month
GET/api/orders/{id}
Single order by ID
GET/api/dashboard/summary
KPI totals — inventory value, low stock, pending orders, backlog
all 4 filters
GET/api/demand
Demand forecasts with trend direction
GET/api/backlog
Backlog items, enriched with has_purchase_order flag
GET/api/spending/summary
Procurement, operational, labor, overhead + YoY
GET/api/spending/monthly
12-month cost breakdown by type
GET/api/spending/categories
Category spend with percentages
GET/api/spending/transactions
Recent transaction list
GET/api/reports/quarterly
Q1–Q4 2025 revenue, fulfillment rates
GET/api/reports/monthly-trends
Month-by-month revenue and order counts
GET/api/tasks
Task list
GET/api/purchase-orders/{backlogItemId}
Purchase orders for a backlog item
