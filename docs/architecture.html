Filter Data Flow

1

User Adjusts Filters

FilterBar.vue exposes four controls: Time Period (month / quarter), Warehouse (San Francisco, London, Tokyo), Category (Circuit Boards, Sensors, Actuators, Controllers, Power Supplies), and Order Status (Delivered, Shipped, Processing, Backordered).

2

Singleton Filter State

All four selections are stored as reactive refs inside useFilters() composable — a module-level singleton shared across all views. Any change triggers watchers in every active view.

3

API Call with Query Params

Each view calls api.js with getCurrentFilters(). Filters are serialised as query parameters, e.g. ?warehouse=London&category=sensors&month=2025-03.

4

Backend Filtering

apply_filters() handles warehouse, category, and status matching. filter_by_month() handles both individual months (2025-01) and quarters (Q1-2025).

5

Pydantic Validation & Response

Filtered Python dicts are validated against Pydantic models (InventoryItem, Order, etc.) and serialised to JSON. The Vue view stores results in ref() and renders via computed properties.
