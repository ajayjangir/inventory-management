Data Flow — Filter Request Lifecycle

1. User selects a filter

FilterBar.vue — v-model binds directly to useFilters reactive refs

2. Shared state updates

useFilters() singleton composable — selectedPeriod, selectedLocation, selectedCategory, selectedStatus

3. View detects change

watch([selectedLocation, selectedCategory, ...]) triggers loadData()

4. API client builds request

api.js — URLSearchParams built from getCurrentFilters(), "all" values skipped

5. Backend receives query params

FastAPI endpoint — optional query params: warehouse, category, status, month

6. In-memory filtering pipeline

apply_filters() for warehouse/category/status, then filter_by_month() for time period (quarters expanded via QUARTER_MAP)

7. Pydantic validates response

Typed models (InventoryItem, Order, etc.) ensure consistent JSON structure

8. Vue updates display

Refs updated → computed properties re-derive display data → template re-renders
